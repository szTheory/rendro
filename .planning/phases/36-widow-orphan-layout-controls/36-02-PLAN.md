<task type="tdd">
  <name>Predictive Text Splitting with Widows/Orphans</name>
  <files>lib/rendro/pipeline/paginate.ex, test/rendro/pipeline/paginate_test.exs</files>
  <behavior>
    - Given a MeasuredText block that exceeds `available_h` (where `available_h = max_h - current_h`):
    - It calculates `line_height_pt = text.height / length(text.lines)` and `lines_fitting = floor(available_h / line_height_pt)`.
    - **Orphans Validation:** If `lines_fitting < text.orphans`, the split is invalid. Push the entire block to the next page.
    - **Widows Validation:** If `total_lines - lines_fitting < text.widows`, reduce `lines_fitting` to satisfy the widow requirement (`lines_fitting = total_lines - text.widows`).
    - **Re-check Orphans:** After reduction, if `lines_fitting < text.orphans`, the split is invalid. Push the entire block to the next page.
    - **Successful Split:** Splits the MeasuredText block into two `Rendro.Block` structs on consecutive pages, with properly adjusted `lines` and `height` based on the line counts. Add an `:info` diagnostic for `:text_split`.
    - **Unfittable:** If `current_h == 0` and it can't split, throw `{:error, :content_overflow, details}` with `line_height_pt` info.
  </behavior>
  <action>
    In `paginate_block/5`, add a pattern match for `%Rendro.Pipeline.MeasuredText{} = text` under `case block.content do`.
    If `current_h + block_h > max_h`, call a new `handle_text_split/10` (analogous to `handle_table_split/10`).
    Inside `handle_text_split/10`, apply the math outlined in the behavior.
    Use `Enum.split(text.lines, lines_fitting)` to get the lines for `this_page_lines` and `remaining_lines`.
    Update `current_page` and `template` with blocks along with an `:info` diagnostic for `:text_split`.
  </action>
