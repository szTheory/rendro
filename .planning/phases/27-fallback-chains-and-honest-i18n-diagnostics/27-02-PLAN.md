  <behavior>
    - Test 1: Text fully covered by the primary font is measured as a single run.
    - Test 2: Text missing a glyph falls back to the secondary font, creating multiple runs.
    - Test 3: Completely missing glyphs (not in any fallback) halt measurement and return `{:error, {:unsupported_glyph, char}}`.
    - Test 4: `Rendro.I18n.Analyzer` is called during block measurement; if unsupported scripts are detected, return `{:error, {:unsupported_script, reason}}`.
  </behavior>
  <action>
    Update `Rendro.Pipeline.Measure`.
    1. Call `FontRegistry.resolve_pdf_font_chain/3` instead of `resolve_pdf_font/3`.
    2. Explicitly apply the 'Core Pattern for Text Measurement' analog from `27-PATTERNS.md`: Hook fallback chain resolution directly into text measurement in `wrap_text` and `split_graphemes` (lines 191-213), preventing blind `Font.text_width` default resolution for unmapped characters.
    3. For each character, find the first font in the chain that has the glyph. Group contiguous characters using the same font into a run `%{font: f, text: str, width: w}`.
    4. If no font has the glyph, do NOT silently fall back to `.notdef`. Instead, trap and halt by propagating an explicit error tuple `{:error, {:unsupported_glyph, char}}`.
    5. Run `Rendro.I18n.Analyzer.analyze/1` on text blocks. If unsupported scripts are present, trap and halt by returning `{:error, {:unsupported_script, reason}}`.
