|----------|-------------|
| caller data map → recipe pagination | Untrusted line count/sizes drive the chunking loop (validated upstream in 74-03 validate_data!/1). |
| recipe → engine paginator | In-process, trusted; recipe pre-chunks ≤ capacity so the engine never re-breaks. |

Pure Elixir library: no network, no auth, no I/O, no persistence, no frontend.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-74-11 | Denial of Service | Pathologically large transaction list driving unbounded pagination/measurement work | accept | Chunking is a single O(rows) pass over the line list (measure once via measure_rows, fold once, chunk once) — linear, no convergence loop (PAGE-04). Cost scales with caller-supplied data, no amplification. A row taller than a whole page is an inherent engine limit (`:content_overflow`, Pitfall 3) surfaced as a typed error, not a hang. |
| T-74-12 | Tampering | Off-by-one chunking tipping a page into hard `:content_overflow` | mitigate | Chunk by the engine's OWN measurement (measure_rows, D-09) and reserve a conservative one-row epsilon margin (D-09/D-10); Task 2 includes a no-overflow test across boundary row counts. |
| T-74-13 | Repudiation | Non-deterministic page breaks / balances undermining audit | mitigate | Deterministic Decimal fold + locale-free `Rendro.Format` + single-pass engine; Task 2 V10 asserts byte-identical render across runs. |
| T-74-14 | Tampering | Stranded/duplicated carried-forward row from engine re-splitting a recipe chunk | mitigate | Each pre-chunked block is ≤ capacity and `keep_together` is never set, so the engine's auto-split never re-fires inside a block (D-10); Task 2 asserts first/last-row labels per page. |
