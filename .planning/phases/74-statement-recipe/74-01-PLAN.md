<objective>
Land the two engine-side enablers the Statement recipe depends on, WITHOUT changing any pagination behavior:

1. Declare `:decimal` as a core dependency in `mix.exs` (D-04). `decimal 2.3.0` is already resolved in `mix.lock` transitively (via `ecto`/`jason`/`jsv` `~> 2.0`), but it is not a direct dep — declaring it lets `%Decimal{}` patterns and `Decimal.*` compile in `lib/`.
2. Expose a thin, read-only PUBLIC measurement helper `Rendro.measure_rows/4` (D-09) that projects the engine's existing private `Measure.measure_block/3` Table-measuring logic so the recipe can chunk transaction rows by the engine's OWN row heights (not a recipe-local estimate that would drift into `:content_overflow`).

Purpose: The recipe must own pagination (D-01) and place carried/brought-forward rows on the correct pages (D-02/D-10). To do that deterministically it needs the engine's actual `header_height` + `row_heights` for a table at a given width. This is the single justified engine touch this phase; it is a pure read.

Output: `:decimal` core dep; `Rendro.measure_rows/4`; a public projection in `measure.ex`; a test asserting the helper matches the engine and is behavior-neutral.
</objective>
