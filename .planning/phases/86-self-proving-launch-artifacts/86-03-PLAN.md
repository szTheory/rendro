  <action>
    Edit `lib/rendro/launch_artifacts.ex`.

    Add a private module attribute named `@launch_table_style` with exactly:
    - `borders: [:outer, :rows]`
    - `border_style: %{color: {216, 210, 195}, width: 0.6}`
    - `header_fill: {247, 243, 234}`

    Add an internal source-document function for launch artifacts, for example `source_document_for/1`, that returns the `%Rendro.Document{}` used for each gallery ID before rendering. The function may be public on the moduledoc-false `Rendro.LaunchArtifacts` module so tests can inspect source fixtures, but it must not be documented as a user-facing recipe API.

    Change `render_source_pdf/1` so each case renders the source document returned by that internal function:
    - `invoice` uses `Rendro.Recipes.Invoice.document(invoice_data())` then applies launch table polish.
    - `branded_invoice` uses `Rendro.Recipes.BrandedInvoice.document(branded_invoice_data())` then applies launch table polish while preserving font and image registries.
    - `statement` uses `Rendro.Recipes.Statement.document(statement_data(45))` then applies launch table polish to every table block generated across pages.
    - `receipt_report` uses `Rendro.Recipes.Receipt.document(receipt_data(58))` then applies launch table polish to every table block generated across pages.
    - `certificate` remains `Rendro.Recipes.Certificate.document(certificate_data(), border: true)` and does not receive table styling.

    Implement table polish as a private traversal over `doc.sections`, `%Rendro.Section{content: ...}`, and `%Rendro.Block{content: %Rendro.Table{}}`. For each table block, update only `borders`, `border_style`, and `header_fill`; preserve every other table, block, section, document, font registry, asset registry, break flag, and measurement field.

    Do not change global table defaults.

    Do not add a broad public recipe styling API.

    Do not change canonical recipe examples in README or guides to imply the launch fixture styling is the default recipe output.
  </action>
