<objective>
Close verification blocker CR-02 and harden the adapter paths that the real snapshot lane now depends on.

`pdfium-render` is an engine-only raster evidence vocabulary. It must remain valid in the top-level `raster.evidence` object, but it must not be structurally valid on GUI-viewer promotion rows such as `forms.viewers.adobe_acrobat_reader`. This plan removes the value from the shared GUI-viewer row schema and validator list, then adds a regression test that fails if the structural boundary is weakened again.

The plan also addresses the directly related raster adapter warnings: page-range argument injection, lexicographic page sorting, and the contradictory `File.rm(path)` before `[:exclusive]` private writes.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| support_matrix.schema.json -> GUI-viewer evidence rows | Schema must reject engine-only raster labels on GUI-viewer promotion rows. |
| validator.ex -> promotion-complete claims | Runtime promotion checks must use the same GUI-viewer-only vocabulary as the schema. |
| caller pages option -> pdfium-cli argv | Page range input is untrusted and must be validated before becoming an argv token. |
| pdfium-cli output files -> returned PNG list | Returned list order is a semantic contract; page_10.png must not sort before page_2.png. |
| private temp input file -> filesystem | Existing files in the private tmp dir are unexpected; writes should fail closed through `:exclusive`. |
