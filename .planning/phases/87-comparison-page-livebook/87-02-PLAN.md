<decision_traceability>
- D-01: Implement the dual-track benchmark model; normalized track feeds quantitative claims and idiomatic track remains fit/DX only.
- D-02: The normalized fixture must be a realistic A4/US Letter invoice or statement with 50-150 rows, repeated headers, footer page numbers, simple borders, fixed fonts, and Latin text only.
- D-03: Idiomatic fixtures must use Rendro data/recipe DSL, ChromicPDF Phoenix/HEEx-style HTML and print CSS, pdf_generator legacy HTML, and Typst native `.typ` template features.
- D-04: Idiomatic observations are non-ranking and must not feed a blended winner chart.
- D-05: Whole process trees count toward RSS/memory metrics.
- D-06: Every committed result publishes pins and environment metadata: versions, Docker image digest, OS, architecture, CPU, memory limit, fonts/assets, fixture, commands, repetitions, median, and p95.
- D-07: Dependency download/build time is excluded from runtime metrics; runtime image size and dependency/runtime count are separate published metrics.
- D-08: ChromicPDF has both cold-start and warm pooled rows.
- D-09: The final manifest/guide must support a job-fit decision guide, not a single overall winner.
- D-11: The manifest contains `schema_version`, `generated_by`, run metadata, scenario metadata, comparator metadata, result rows, public claim records, and SHA-256 raw artifact refs.
</decision_traceability>

    - 50 to 150 line items;
    - totals;
    - Latin text only;
    - no complex scripts, CSS grid/flex showcase, math typesetting, or web-template migration content.

    Create four normalized fixture implementations:
    - `invoice_rendro.exs` builds `Rendro.Recipes.Invoice.document/1` from `invoice_data.json` and renders deterministic PDF bytes.
    - `invoice_chromic_pdf.html.eex` renders the same data as plain HTML plus print CSS for ChromicPDF.
    - `invoice_pdf_generator.html.eex` renders the same data as legacy-friendly HTML for pdf_generator.
    - `invoice_typst.typ` consumes JSON or `--input` data through Typst's `sys.inputs` and renders an equivalent native Typst document.

    Create `bench/comparison/pins.json` with explicit keys for:
    - `rendro` version from `Mix.Project.config()[:version]`;
    - `chromic_pdf` package constraint and resolved version;
    - `pdf_generator` package constraint and resolved version;
    - `typst_cli` version and binary source;
    - Chrome/Chromium version when used by ChromicPDF;
    - wkhtmltopdf version when used by pdf_generator;
    - Docker base image tag and digest;
    - fonts/assets used by all fixtures.

    Do not add these comparator libraries or external tools to Rendro runtime dependencies.
  </action>
