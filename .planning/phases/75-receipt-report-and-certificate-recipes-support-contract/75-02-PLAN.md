| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-75-02-01 | Spoofing | totals.total assertion | mitigate | maybe_validate_totals!/2 validates total == subtotal + tax - discount using Decimal.equal?/2; caller cannot silently supply wrong total |
| T-75-02-02 | Denial of Service | very large :lines list (e.g. 100,000 items) | accept | chunk_rows_into_pages/2 is O(n); Rendro.measure_rows/4 is the expensive call (one measure per row); no infinite loop; at extreme scale, memory use is proportional to input — acceptable for a library function where the caller controls input size |
| T-75-02-03 | Tampering | float amount bypassing Decimal.equal? check | mitigate | validate_data!/1 raises ArgumentError for any line.amount that is not a Decimal struct (type check, same pattern as Statement) |
| T-75-02-04 | Denial of Service | totals block on last page causing :content_overflow | mitigate | totals block is small (2–4 rows equivalent); recipe reserves @row_epsilon margin; effective_capacity formula leaves slack; V9 test at boundary row counts proves no overflow |
