<objective>
Refine the PNG parsing utility to handle data contracts for color spaces and palettes, and explicitly reject interlaced PNGs. Also establish the phase validation criteria.

Purpose: To ensure the PNG to PDF conversion is robust and handles all required PNG types (or rejects them safely). Specifically addresses gaps in color space handling, palette extraction, and interlaced image rejection.
Output: Refined `Rendro.PDF.PNG` module, comprehensive test suite, and `30-VALIDATION.md`.
</objective>

<feature>
  <name>PNG Data Contract Refinement and Rejection</name>
  <files>lib/rendro/pdf/png.ex, test/rendro/pdf/png_test.exs</files>
  <behavior>
    - `decode/1`: Returns `{:ok, %{...}}` or `{:error, :interlaced}`.
    - `process_for_pdf/1`: Returns:
        - `{:pass_through, idat_binary, decode_parms, color_space}`
        - `{:split, color_binary, alpha_binary, color_space}`
    - `color_space` should be:
        - `{:name, "DeviceRGB"}` for RGB
        - `{:name, "DeviceGray"}` for Gray
        - `[:Indexed, {:name, "DeviceRGB"}, max_index, palette_binary]` for Indexed images (using PLTE).
    - Adam7 (interlaced) PNGs must be rejected with `{:error, :interlaced}`.
  </behavior>
  <implementation>
    Update `lib/rendro/pdf/png.ex`.
    Modify `decode/1` to return `{:ok, map}` or `{:error, :interlaced}`.
    Implement `process_for_pdf/1` that uses `decode/1` and `decode_pixels/1`.
    For Indexed PNGs (color_type 3), extract `PLTE` chunk and return the `/Indexed` array format.
    Ensure `IDAT` pass-through only happens when no alpha channel is present.
  </implementation>
</feature>
