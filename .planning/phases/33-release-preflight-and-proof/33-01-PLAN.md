# Phase 33: Release Preflight and Proof

## Context & Strategy

We need to verify the release artifacts and dry-run the Hex publication process. Following Elixir idiomatic practices, we want preflight checks to be deterministic and runnable offline, meaning our LLM-backed visual UAT must remain out of the standard `mix release.preflight`. However, we want strict programmatic guarantees that the exact files promised by our documentation contract are present in the final Hex artifact.

**Decisions Made:**
1. **Visual UAT:** Do NOT add `mix rendro.visual_uat` to `mix release.preflight` or CI aliases. It requires external binaries (`pdftoppm`) and authenticated API keys (Anthropic), which violates the principle of least surprise for OSS contributors. We will run it manually as a final check.
2. **Hex Integrity Check:** We will extend `mix release.preflight` with a new check: `check_hex_artifacts`. It will run `mix hex.build --unpack`, inspect the generated `rendro-<version>/` directory, and strictly assert the presence of `LICENSE`, `README.md`, `CHANGELOG.md`, and all `guides/*.md`.

## Task 1: Add Hex Artifact Assertion to Preflight

**Goal:** Prove the Hex package physically contains what we claim it contains.

1. Open `lib/mix/tasks/release/preflight.ex`.
2. Remove `{"Hex Build Unpack", ["hex.build", "--unpack"]}` from `@phase_2_checks` (we will do this in Phase 1 instead so we can assert on its contents).
3. Add a new check to `phase_1_results` called `check_hex_artifacts(context, version)`.
4. The new `check_hex_artifacts/2` function will:
   - Run `run_command(context, "mix", ["hex.build", "--unpack"])`.
   - Assert that the directory `rendro-#{version}` exists.
   - Assert that the following files exist inside `rendro-#{version}/`:
     - `LICENSE`
     - `README.md`
     - `CHANGELOG.md`
     - `guides/api_stability.md`
     - `guides/branding.md`
     - `guides/integrations.md`
   - Return pass/fail accordingly.
   - Clean up the directory (`File.rm_rf!("rendro-#{version}")` and `File.rm("rendro-#{version}.tar")`).

## Task 2: Verify Final Release Surface

**Goal:** Prove everything passes.

1. Run `mix rendro.visual_uat` manually to confirm the latest UI changes haven't broken the branded PDF.
2. Run `mix release.preflight` to execute the full CI suite, docs contract, the new Hex artifact assertion, and the `hex.publish --dry-run`.
3. If both pass, Phase 33 is complete.
