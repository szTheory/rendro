# Phase 81: Release Hardening & Repo Hygiene

**Objective**
Ensure the repository, GSD state, and GitHub CI are in a mathematically clean and pristine state before cutting the 1.0.0 Hex release, and codify this discipline into an automated bash script to guarantee release hygiene going forward.

**Key Files & Context**
- `scripts/repo_hygiene_check.sh` (New)
- `.github/workflows/ci.yml`
- `.github/workflows/release.yml`
- Local Git Worktrees and GSD state

**Implementation Steps**

1. **Local & Remote Git Cleanup:**
   - Execute `git worktree prune` to clear out stale or dangling worktrees.
   - Run `git status` to ensure a completely clean working directory.
   - Ensure `main` is up-to-date with remote and all open PRs are triaged, merged, or safely closed.

2. **GSD State Consolidation:**
   - Verify there are no orphaned `.planning/HANDOFF.json` or debug files.
   - Ensure all specs, lint checks, and formatting rules pass strictly via the existing `mix ci` alias.

3. **Automate the Discipline (Shift-Left):**
   - Create a dedicated `scripts/repo_hygiene_check.sh` bash script to mechanically enforce hygiene:
     - Verifies the git working tree is clean (`git diff --quiet` and `git diff --cached --quiet`).
     - Ensures there are no untracked files in the critical paths.
     - Automatically invokes `mix ci` to run the full gauntlet (format, compile, test, docs, credo, dialyzer).
   - Ensure the script is made executable (`chmod +x`).

4. **Phase 81 Hardening (Release Pipeline):**
   - Update `.github/workflows/ci.yml` and `release.yml` to SHA-pin all third-party GitHub Actions (e.g., `actions/checkout@v4` becomes `actions/checkout@<sha>`).
   - Resolve any CHANGELOG self-blocking issues (as defined in the Phase 81 roadmap goal) to clear the path for the future Hex publish.

**Verification & Testing**
- Run `scripts/repo_hygiene_check.sh` locally and ensure it exits with `0`. 
- Temporarily modify a tracked file and verify the script correctly fails and aborts.
- Confirm GitHub Actions CI remains green after SHA-pinning the workflows.
