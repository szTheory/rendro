  <files>lib/rendro/fragmentable.ex, test/rendro/link_test.exs</files>
  <behavior>
    - A link whose inner content fragments yields one link wrapper for the fitting fragment and one for the remaining fragment.
    - Both fragments preserve the same validated URI or page target.
    - Fragment semantics stay rectangular and block-based rather than emitting line-level or glyph-level hit regions.
  </behavior>
  <action>
    Add a `Rendro.Fragmentable` implementation for the new link node so splitting delegates to the wrapped content and rewraps both sides with the same target per D-15.

    Extend `test/rendro/link_test.exs` with fragment-focused cases that prove multi-page text links produce two link wrappers carrying the same target and that unsplittable content still respects the existing `{nil, component}` behavior.
  </action>
  <verify>
    <automated>mix test test/rendro/link_test.exs</automated>
  </verify>
  <done>Paginated fragments preserve the same link target and remain aligned with the existing block-based fragmentation model.</done>
