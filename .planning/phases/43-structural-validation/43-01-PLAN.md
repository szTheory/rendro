# Plan 43-01: Hybrid Single-Pass AST Validation

## Goal
Implement a robust, performant validation stage that walks the PDF AST once and applies decoupled validation rules.

## Tasks

1. **Implement Core Walker (`Rendro.Pipeline.Validate`)**
   - Create `lib/rendro/pipeline/validate.ex`.
   - Implement the recursive tree traversal over the AST document struct.
   - Configure a list of rules to apply at each node.

2. **Implement Validation Rules (`Rendro.Rules.*`)**
   - Implement `CheckReferences`: Validates that any object IDs referenced by `Kids`, `Contents`, etc., actually exist in the Document.
   - Implement `CheckBounds`: Ensures all layout bounding boxes are valid structures.
   - Implement `CheckRequiredKeys`: Checks that critical PDF dictionary keys (e.g., `Type`) are present for specific object types.

3. **Integrate into Pipeline**
   - Ensure the new `Validate` stage runs after `Paginate` and before `Render` (binary serialization).

4. **Validation and Stress Testing**
   - Write unit tests for individual rules.
   - Write an integration stress test that generates a large, deeply nested AST and passes it through `Validate` to prove correctness and performance.
