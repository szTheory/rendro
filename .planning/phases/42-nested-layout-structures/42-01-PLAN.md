# Plan 42-01: Protocol-Based Nested Fragmentation

## Goal
Implement the `Rendro.Fragmentable` protocol and refactor `Paginate` to use it, enabling flawless deterministic pagination for deeply nested tables and blocks.

## Tasks

1. **Define `Rendro.Fragmentable` Protocol**
   - Create `lib/rendro/fragmentable.ex`.
   - Define the `split(component, available_height)` function contract.

2. **Implement Protocol for `Rendro.Block`**
   - Implement `Fragmentable` for `%Rendro.Block{}`.
   - If the block fits, return `{:fit, block}`.
   - If it doesn't fit, recursively call `Fragmentable.split` on its inner `content` (which could be `MeasuredText`, `Table`, etc.).
   - Handle the returned fragments and reconstruct `{fitting_block, remaining_block}`.

3. **Implement Protocol for `Rendro.Table`**
   - Implement `Fragmentable` for `%Rendro.Table{}`.
   - Iterate through rows to find where the page boundary is crossed.
   - Split the crossing row (by splitting its cells/blocks).
   - Reconstruct `{fitting_table, remaining_table}` ensuring headers are repeated if configured, and borders respect `decoration_break`.

4. **Refactor `Rendro.Pipeline.Paginate`**
   - Update the pagination engine to rely on `Rendro.Fragmentable.split/2`.
   - Remove hardcoded logic for text splitting in favor of the new protocol.

5. **Validation and Stress Testing**
   - Write comprehensive tests in `test/rendro/pipeline/paginate_nested_test.exs`.
   - Ensure a Table nested inside a Table splits cleanly and renders valid AST.
