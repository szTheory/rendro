Adapter error contracts to document (post 05-02 + 05-03):

Mailglass — after 05-03 fixes:
- `{:error, %Rendro.Error{reason: {:invalid_email_target, term()}}}` — caller passed something that is neither a `%Swoosh.Email{}` nor a recognized Mailglass message struct.
- `{:error, {:unrecognized_message_shape, atom()}}` — caller passed a Mailglass.* message struct (or any struct ending in `.Message`) that has neither `:swoosh` nor `:email`.
- `{:error, %Rendro.Error{}}` — propagated from `Rendro.render/1` when render itself fails (max_pages, max_bytes, timeout, etc.).

Threadline — current behavior:
- `:ok` — `Threadline.record_action/2` returned `:ok` or `{:ok, _}`.
- `{:error, _}` — `Threadline.record_action/2` returned `{:error, _}`.
- `{:error, {:unexpected_return, term()}}` — `Threadline.record_action/2` returned something other than `:ok`/`{:ok, _}`/`{:error, _}`.
- `{:error, {:exception, Exception.t()}}` — `Threadline.record_action/2` raised; the adapter rescues and wraps it.
- `<no audit row>` — Pipeline timeouts (WR-01): `Pipeline.run/1` shuts the Task down before its `:telemetry.span` can emit `:stop`/`:exception`, so timed-out renders return `{:error, %Rendro.Error{reason: :timeout}}` to the caller but produce zero Threadline calls. **Known limitation, NOT fixed in this gap-closure run.**

Accrue — after 05-02:
- `{:ok, %Rendro.Document{}}` — successful recipe.
- `{:error, {:invalid_invoice, term()}}` — input was not an `%Accrue.Invoice{}`.
- (Render-time errors flow through `Rendro.render/1`, not the recipe — `{:error, %Rendro.Error{}}` from any of `:build`, `:compose`, `:measure`, `:paginate`, `:render` stages.)
