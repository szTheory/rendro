From lib/rendro/pipeline/measure.ex — CURRENT split_graphemes (lines 601–650):
```elixir
defp split_graphemes(text, max_width, font_chain, font_size) do
  result =
    Enum.reduce_while(String.graphemes(text), {:ok, {[], []}}, fn grapheme,
                                                                  {:ok, {lines, current_line}} ->
      case find_font_for_grapheme(grapheme, font_chain) do
        {:ok, font} ->
          {:ok, glyphs} = Rendro.Text.Shaper.shape(font, grapheme)  # BUG: shapes per-grapheme
          width = glyphs |> Enum.reduce(0, fn g, acc -> acc + g.x_advance end)
                  |> Kernel.*(font_size / font.units_per_em)
          grapheme_run = [%{font: font, text: grapheme, width: width}]
          candidate_line = merge_runs(current_line, grapheme_run)
          cond do
            current_line == [] and width <= max_width ->
              {:cont, {:ok, {lines, candidate_line}}}
            current_line == [] ->
              {:cont, {:ok, {[candidate_line | lines], []}}}
            runs_width(candidate_line) <= max_width ->
              {:cont, {:ok, {lines, candidate_line}}}
            true ->
              {:cont, {:ok, {[current_line | lines], grapheme_run}}}
          end
        :error ->
          {:halt, {:error, {:unsupported_glyph, grapheme}}}
      end
    end)
  ...
end
```

Post-Plan-03, the line `{:ok, glyphs} = ...` has already been changed to a case expression. The cluster-boundary fix in this plan changes the shaping strategy — instead of shaping each grapheme individually, the function should accumulate graphemes into a run, shape the run, then use glyph cluster fields to determine break points.

The glyph struct from Shaper.Simple (Plan 01 Task 1) has `cluster: 0` for all glyphs — this means Shaper.Simple's output cannot use cluster-based breaking directly. The cluster-boundary behavior is only meaningful for the HarfBuzz adapter path. For Shaper.Simple, the current per-grapheme loop IS the cluster-boundary behavior by construction (each grapheme = one cluster).

D-11 says "uniform fix, no two-path fork" and "run-shaping is byte-identical on the pure-Elixir path by construction." The fix: instead of shaping grapheme-by-grapheme, shape the full candidate segment at once and use the returned glyph list with cluster fields for boundary decisions. With Simple (cluster=0 for all), this collapses to the same behavior. With HarfBuzz, actual cluster values guide where breaks are safe.

```elixir
property "per-grapheme width sum equals per-run width under Shaper.Simple" do
  check all text <- StreamData.string(:ascii, min_length: 1) do
    font = ... # use a standard built-in test font
    {:ok, per_run_glyphs} = Rendro.Text.Shaper.Simple.shape(font, text, [])

    per_grapheme_total =
      text
      |> String.graphemes()
      |> Enum.map(fn g ->
        {:ok, glyphs} = Rendro.Text.Shaper.Simple.shape(font, g, [])
        Enum.sum(Enum.map(glyphs, & &1.x_advance))
      end)
      |> Enum.sum()

    assert per_run_total == per_grapheme_total
  end
end
```
