<objective>
Ship `Rendro.Branded` — a tiny public helper module exposing two functions:
- `Rendro.Branded.font_path/0` → absolute path to `priv/branded/fonts/B612-Regular.ttf`
- `Rendro.Branded.logo_path/0` → absolute path to `priv/branded/images/rendro-logo.png`

Both wrap `Application.app_dir(:rendro, "priv/branded/...")` exactly. No fallback, no caching, no `File.exists?/1` checks inside the function bodies, no system-font discovery. The module's `@moduledoc` explicitly documents these as "demo assets — NOT a built-in font or default logo" per D-11.

Also ships `test/rendro/branded_test.exs` — a 3-test ExUnit module that asserts `font_path/0` and `logo_path/0` return binaries pointing at existing files (the bytes committed in Plan 01).

This plan implements D-10, D-11, D-31 from CONTEXT.md.

Purpose: Provide the canonical path-resolution API that Plan 03's `Rendro.Recipes.BrandedInvoice.document/2` will call internally. Same call works in `mix test`, `iex -S mix`, doctests, and downstream Hex consumer apps — same convention as `tzdata`, `gettext`, `cldr_*`.

Output: One small public helper module + matching unit test. Total LOC budget: ~30 lines of `lib/` code + ~25 lines of test code.
</objective>
