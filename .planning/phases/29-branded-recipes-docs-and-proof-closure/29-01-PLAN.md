    Step 1 — Create `scripts/render_logo.exs` as a pure-Erlang/`:zlib` PNG encoder. Match the `scripts/release_preflight_proof.exs` convention:

    The script's job is to deterministically emit a 64x64 RGBA PNG into `priv/branded/images/rendro-logo.png` using only Erlang's stdlib `:zlib` and `:erlang.crc32/1`. Skeleton (per RESEARCH.md lines 651-672, adapted):

    ```elixir
    defmodule Rendro.RenderLogo do
      @moduledoc false

      def run do
        png_signature = <<137, 80, 78, 71, 13, 10, 26, 10>>

        # IHDR: width=64, height=64, bit_depth=8, color_type=6 (RGBA), compression=0, filter=0, interlace=0
        ihdr_data = <<64::32, 64::32, 8, 6, 0, 0, 0>>
        ihdr = chunk("IHDR", ihdr_data)

        raw = build_pixels()
        idat = chunk("IDAT", :zlib.compress(raw))
        iend = chunk("IEND", <<>>)

        png = png_signature <> ihdr <> idat <> iend
        File.write!("priv/branded/images/rendro-logo.png", png)
        IO.puts("Wrote priv/branded/images/rendro-logo.png (#{byte_size(png)} bytes)")
      end

      # Build 64 scanlines of 64 RGBA pixels, each prefixed with PNG filter byte 0 (None).
      # Aesthetic: simple geometric mark — solid background with a centered shape.
      # The agent has discretion on exact pixel composition (CONTEXT.md "Claude's Discretion" item:
      # "Exact bytes of the hand-authored 64×64 PNG logo (must be deterministic and committed; aesthetic
      # is the agent's call within the byte budget)"). Keep it simple to maximize zlib compression
      # ratio (large flat regions compress well — keeps total well under 2_000 bytes).
      defp build_pixels do
        # Example: dark teal background with a centered light circle.
        # Choose colors as deterministic literals; do NOT use random/time-based generation.
        bg = <<13, 71, 76, 255>>     # dark teal RGBA
        fg = <<240, 240, 240, 255>>  # near-white RGBA

        for y <- 0..63, into: <<>> do
          row =
            for x <- 0..63, into: <<>> do
              # Centered circle, radius ~22, distance squared check
              dx = x - 32
              dy = y - 32
              if dx * dx + dy * dy < 22 * 22, do: fg, else: bg
            end

          <<0>> <> row  # PNG filter byte 0 (None) prefix per row
        end
      end

      defp chunk(type, data) when byte_size(type) == 4 do
        crc = :erlang.crc32(type <> data)
        <<byte_size(data)::32, type::binary, data::binary, crc::32>>
      end
    end

    Rendro.RenderLogo.run()
    ```

    The aesthetic above (dark-teal disc on lighter background) is one valid choice; the agent MAY swap colors or substitute another simple geometric shape (square inset, ring outline, two-tone diagonal) so long as the resulting bytes are deterministic AND the compressed PNG is < 2_000 bytes AND it remains 64x64 RGBA. The exact bytes are explicit "Claude's Discretion" per CONTEXT.md. KEEP IT SIMPLE — flat color regions compress well.
