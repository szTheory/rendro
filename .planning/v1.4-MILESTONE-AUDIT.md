# Milestone v1.4 Audit

**Milestone:** Async Delivery and Artifact Operations
**Date:** 2026-05-05

## Summary

The `v1.4` milestone successfully implemented queued render lifecycle, artifact metadata, and persistence/sink contracts. It added robust external integrations for Accrue, Mailglass, and Oban to provide production-grade async workflows. Additionally, it delivered advanced table fragmentation DSL, cell fragmentation, and grid projection, ensuring complex flow layouts function deterministically.

## Completed Work

### Core Infrastructure
- Introduced `Rendro.Artifact` for encapsulating rendered binaries alongside their deterministic hashes and metadata.
- Added `Rendro.render_to_artifact/2` to standardize the artifact lifecycle boundary.
- Defined `Rendro.Storage` and `Rendro.Audit` behaviors for externalizing persistence and telemetry logging.

### Integrations
- Shipped the `Accrue` adapter for integrated billing document workflows.
- Shipped the `Mailglass` adapter for seamless transactional email attachments.
- Shipped `Oban.RenderWorker` as the canonical asynchronous job runner for background rendering.

### Layout & Pagination
- Implemented table fragmentation DSL.
- Built grid projection logic in the Measure phase to appropriately bound table cells in memory.
- Finalized explicit cell fragmentation in the Paginate phase to break complex rows correctly across pages.

## Verification
- Integrations and layout enhancements were verified via unit tests, matching deterministic snapshots.
- Core regression suites for previously working features (typography, layouts) continue to pass.
- `mix ci` and `mix verify` are fully operational.

## Path Forward

With `v1.4` shipped, Rendro now has mature async and integration boundaries. The next focus (v1.5) will shift to Operational Tooling & Admin UX, addressing feature flagging, secure workflows, and rollout governance (`Rulestead`, `Sigra`).
