# Milestone v2.0: Signature Fields & External Signing Preparation

**Status:** ✅ SHIPPED 2026-05-07
**Started:** 2026-05-06
**Phases:** 55-59
**Total Plans:** 8

## Overview

`v2.0` adds a narrow signing-preparation story to Rendro by extending the existing authored forms surface with unsigned signature fields, keeping the core writer deterministic, and handing final artifacts into explicit external-signing workflows. The milestone is intentionally scoped to field authoring, signing preparation, and truthful support boundaries; cryptographic signing and compliance claims remain deferred. Phases 58 and 59 close the audit trail by backfilling the missing Phase 55 and 56 verification artifacts without rewriting shipped implementation history.

## Phase Ordering Rationale

- Phase 55 constrains public semantics first so later writer work only handles supported field shapes.
- Phase 56 follows because signing-preparation workflows depend on final rendered bytes and deterministic object wiring.
- Phase 57 closes the public support boundary because this surface is trust-sensitive and easy to overclaim without dedicated proof and docs discipline.
- Phases 58 and 59 preserve shipped-history truth by backfilling the missing verification artifacts rather than pretending the original implementation phases were reimplemented.

## Deferred

- In-core cryptographic signing
- First-party certificate or key-management workflows
- PAdES/LTV/TSA/OCSP/CRL proof and compliance narratives
- Broad viewer support claims without recorded evidence

## Milestone Summary

**Key Decisions:**

- Kept unsigned signature authoring on the shared `%Rendro.FormField{}` seam through `Rendro.signature_field/2` instead of opening a parallel forms engine.
- Kept ordinary render output strictly unsigned by emitting visible `/Sig` widgets without `/V`, `/ByteRange`, signer-owned `/Contents` placeholders, `/Lock`, `/SV`, `/Filter`, or `/SubFilter`.
- Kept `Rendro.Sign.prepare/2` artifact-first over final bytes only, with a narrow shared `metadata.signing_preparation` manifest and isolated adapter-local metadata.
- Kept public support language split across authored helper, unsigned widget, and signing-preparation claims so digital-signature, viewer-validity, and compliance narratives remain explicitly unsupported.
- Accepted verification-backfill phases as the truthful way to restore audit-grade closure when implementation shipped before the per-phase verification artifacts existed.

**Issues Resolved:**

- Added explicit authored unsigned signature-field support and validate-stage rejection for scope-breaking signing metadata.
- Added deterministic unsigned signature widget serialization and repeated-render proof for identical authored inputs.
- Added artifact-first external-signing preparation with deterministic placeholder offsets and typed failure handling.
- Aligned the support matrix, guide wording, docs-contract lanes, and verification notes around one canonical unsigned/preparation vocabulary.
- Closed the milestone audit gap by backfilling `55-VERIFICATION.md` and `56-VERIFICATION.md` and re-closing all nine requirements through explicit proof.

**Issues Deferred:**

- Cryptographic signing, key custody, and certificate management remain outside core and outside this milestone.
- PAdES/LTV/TSA/OCSP/CRL and broad compliance claims remain deferred to a later proof-backed milestone.
- Signature-specific viewer promotion remains `unverified` until exact per-viewer evidence is recorded.

**Technical Debt Incurred:**

- No `v2.0` release-blocking debt remains in the milestone artifact trail after the Phase 58/59 backfill work.
