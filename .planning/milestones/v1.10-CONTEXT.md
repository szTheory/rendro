# Milestone Context: v1.10 Protected Delivery Hooks & Encryption Boundaries

**Created:** 2026-05-06
**Status:** Active

## Goal

Add a truthful, artifact-first PDF protection story that gives Phoenix teams a practical password-to-open delivery path without destabilizing Rendro's deterministic core or overclaiming viewer/security behavior.

## Locked Scope

- External protection hooks first
- Artifact-first public API
- AES-256 only
- Advisory permissions explicitly labelled as advisory
- Existing delivery seams continue to transport already-protected artifacts
- Release-preflight tail included so the next Hex publish can happen right after proof closes

## Non-goals

- Native in-core encryption
- Legacy algorithms
- Persisting protection secrets in async job args
- Digital signatures, tamper evidence, or compliance/archive claims
- Blanket viewer compatibility statements

## Why Now

`v1.9` extended Rendro's authored PDF surface with embedded files and curated links. The next high-value step for downstream usage, especially `mailglass`, is a narrow protected-delivery story that still respects the library's deterministic and truthful-contract DNA.
