# Requirements: Rendro v1.8

**Defined:** 2026-05-05
**Shipped:** 2026-05-05
**Core Value:** Phoenix teams can generate reliable, auditable, deterministic PDFs from Elixir data/components, with clear pagination behavior and production-grade observability.

## v1.8 Requirements

### Interactive Widget Authoring

- [x] **FORMS-01**: Engineers can author deterministic text form fields through Rendro primitives, including default values and explicit geometry.
- [x] **FORMS-02**: Engineers can author deterministic checkbox and radio widgets through the same public form-field surface, with explicit group and export semantics for radios.

### Validation and Serialization

- [x] **FORMS-03**: Core validation rejects unsupported, contradictory, or ambiguous authored form state before render.
- [x] **FORMS-04**: The PDF writer emits AcroForm catalogs, widget annotations, and deterministic appearance streams without relying on viewer-generated appearances.

### Truthful Support Boundaries

- [x] **FORMS-05**: Public docs and the machine-readable support matrix publish the same proof-backed forms support contract, including explicit viewer status.

## Requirement Outcomes

All five `v1.8` requirements shipped and remain traced to committed implementation and verification artifacts.
