# Milestone Roadmap: Bulletproof Core & Feature Completeness (v1.5)

## Vision
Transform Rendro's pure-Elixir core into a bulletproof, feature-complete layout engine capable of rendering complex, deeply nested business documents (tables within tables) with precise pagination controls (widows/orphans), all verified by native structural PDF invariants.

## Success Criteria
- Nested tables and text blocks correctly break across pages without layout violations.
- A native validation stage proves structural soundness before binary serialization.

## Key Risks
- **Performance:** Recursive AST fragmentation and deep tree traversal could impact render speeds for massive documents.
- **Protocol Boundary:** Ensuring `Rendro.Fragmentable` cleanly separates content chunks without losing semantic borders or styling flags.

## Proof Strategy
- **Stress-Test Fixtures:** A dedicated "extreme report" fixture containing deep nesting and severe break constraints.
- **Validation Pipeline Stage:** A new validation pass ensures the resulting PDF AST contains valid object references and bounds.

## Verification Classes
- Deterministic Output (Byte-level or structured AST output)
- Visual UAT (Rendered PDF checks)
- Invariant Testing (Internal PDF structure checks)

## Definition of Done
- All slices are complete.
- `docs.contract` tests run successfully against the new features.
- No performance regressions beyond the agreed budget.

## Requirement Coverage
- Addresses v1.5 Epic goal: Validation and Trust Surfaces.
- Addresses core layout maturity for complex reporting SaaS use cases.

## Boundary Map
- `S01 (Widow/Orphan)` → Upgrades `Rendro.Pipeline.Paginate` to respect predictive line splitting. Consumed by later nesting.
- `S02 (Table Fragmentation)` → Introduces `Rendro.Fragmentable` to `Rendro.Table`.
- `S03 (Nested Layouts)` → Fuses `S01` and `S02` logic recursively.
- `S04 (Validation)` → Introduces `Rendro.Pipeline.Validate` taking the final AST produced by `S03`.

## Slices

- [ ] **S01: Widow/Orphan Layout Controls** `risk:medium` `depends:[]`
  > After this: Multi-line text blocks with widow/orphan limits break cleanly across page boundaries without leaving solitary lines.
- [ ] **S02: Advanced Table Continuation & Explicit Cell Fragmentation** `risk:high` `depends:[]`
  > After this: Multi-page tables strictly follow explicit cell fragmentation rules, keeping content bound properly across pages.
- [ ] **S03: Nested Layout Structures** `risk:high` `depends:[S02]`
  > After this: A table nested inside another layout component aggregates heights correctly and fragments cleanly if it crosses a page boundary.
- [ ] **S04: Structural PDF Validation & Stress Test** `risk:medium` `depends:[S01, S03]`
  > After this: A generated stress-test PDF passes a native structural validation pass to prove layout soundness.
