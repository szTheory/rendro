# Milestone v1.6: Async Delivery and Artifact Operations Roadmap

## Vision
Transform Rendro from a pure layout engine into a production-ready async document pipeline for SaaS environments, utilizing a strict behavior-driven adapter pattern to integrate with Oban, standard storage, and the `szTheory` ecosystem without compromising the pure core.

## Success Criteria
- SaaS operators can queue background rendering jobs reliably via Oban.
- Rendered PDFs produce deterministic artifacts with structured metadata.
- Render successes and structured failures are durably logged to `threadline`.
- Artifacts can be seamlessly attached to `mailglass` transactional emails.
- The `rendro` hex package retains zero hard dependencies on database, queueing, or network I/O libraries.

## Key Risks
- **Core Contamination**: Side-effects creeping into the `Rendro.Core` engine, destroying determinism and testability.
- **Opaque Failures**: Layout errors being lost in async queues, causing silent failures for operators.
- **Non-deterministic Retries**: Oban retries producing different PDF hashes due to ambient timestamps.

## Proof Strategy
We will build incrementally from the pure core outward. First, we define the pure data carrier (`Rendro.Artifact`). Then, we define the side-effect boundaries (`Storage` and `Audit` behaviors). Next, we implement the async queue driver (`Oban` worker). Finally, we build the external ecosystem adapters (`threadline`, `mailglass`, `accrue`) that rely on those boundaries.

## Verification Classes
- **Data Integrity**: Unit tests for `Rendro.Artifact` metadata extraction and hashing.
- **Contract Fulfillment**: Integration tests ensuring optional adapters (e.g., Oban worker) strictly fulfill defined Behaviors without calling core internals improperly.
- **System Integration**: End-to-end simulated run passing a generated document through the Oban queue, mock storage, and `threadline` audit adapter.

## Definition of Done
- All slices completed and verified.
- No mandatory dependencies on Ecto, Oban, or AWS added to the main project `mix.exs`.
- Documentation updated with clear canonical recipes for all integrations.
- Code conforms to existing `mix format` and `mix credo --strict` standards.

## Requirement Coverage
- Supports EPIC.md requirement: "Async Delivery and Artifact Operations".
- Fulfills user needs for robust operationalization of PDF generation.
- Covers "Do Now" integrations: `threadline`, `mailglass`, `accrue`.
