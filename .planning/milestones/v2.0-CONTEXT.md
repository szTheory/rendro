# Milestone Context: v2.0 Signature Fields & External Signing Preparation

**Created:** 2026-05-06
**Status:** Shipped 2026-05-07

## Goal

Add a narrow, truthful signing-preparation surface that lets Phoenix teams author unsigned signature fields and prepare rendered artifacts for external signing without turning Rendro core into a cryptographic trust or compliance stack.

## Locked Scope

- Unsigned signature-field authoring only if it fits the existing authored form model truthfully
- Deterministic writer support for signature widgets and related AcroForm structures
- Artifact-first preparation seams for external signing after render output exists
- Truthful support-matrix, docs-contract, and guide wording for signature preparation boundaries

## Non-goals

- In-core cryptographic signing
- Core key custody or certificate management
- PAdES/LTV/TSA/OCSP/CRL and blanket compliance claims
- Blanket viewer compatibility statements without recorded proof
- Reframing protection or qpdf support as digital-signature support

## Why Now

`v1.8` established Rendro's authored AcroForm surface, and `v1.10` established the artifact-first pattern for trust-sensitive post-processing. `v2.0` shipped that bridge as a truthful signing-preparation story without widening the core render pipeline or abandoning Rendro's proof-backed support posture.
