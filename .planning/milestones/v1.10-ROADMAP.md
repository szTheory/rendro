# Milestone v1.10: Protected Delivery Hooks & Encryption Boundaries

**Status:** ✅ SHIPPED 2026-05-06
**Started:** 2026-05-06
**Phases:** 51-54
**Total Plans:** 8

## Overview

`v1.10` adds a narrow, truthful PDF protection story through an artifact-first external hook instead of widening the core render pipeline. The milestone is intentionally scoped around AES-256 password-to-open protection, advisory-permission honesty, password-safe adapter boundaries, proof-backed support language, and a release tail that prepares the next Hex publish for downstream `mailglass` usage.

## Key Decisions

- Protection remains post-render and artifact-first rather than becoming a new pipeline stage.
- AES-256 is the only supported public algorithm for this milestone.
- Viewer rows remain `unverified` until recorded evidence exists; structural validation alone is not viewer proof.
- Existing delivery seams transport protected artifacts but do not receive or persist password material themselves.

## Deferred

- Native in-core encryption
- Signature fields and external signing preparation
- Compliance/archive/tamper-evidence claims

## Milestone Summary

**Key Decisions:**

- Protection shipped as an artifact-first external hook rather than a new core pipeline stage.
- AES-256 is the only supported public protection algorithm for this milestone.
- Viewer support stayed evidence-driven: Apple Preview was promoted only after a full recorded checklist; Acrobat remains `unverified`.
- Release readiness now fails early if the current changelog loses the canonical protected-delivery pointer.

**Issues Resolved:**

- Added a first-party `qpdf` adapter with runtime-optional executable lookup and typed, redacted failures.
- Extended the Poppler validation lane so protected PDFs can be structurally verified with the right password.
- Preserved existing delivery/storage seams for already-protected artifacts without persisting password material.
- Closed the release tail with proof-backed docs, support-matrix sync, and executable preflight gates.

**Issues Deferred:**

- Adobe Acrobat Reader viewer proof for the `protection` surface remains intentionally unverified until recorded manually.
- Native in-core encryption, digital signatures, and compliance-oriented trust claims remain future work.

**Technical Debt Incurred:**

- No release-blocking debt remained at ship time; formatting, Dialyzer, and tagged-proof issues were closed before publish readiness was declared.
