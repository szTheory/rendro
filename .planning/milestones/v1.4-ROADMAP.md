# Milestone: Advanced Layout & Typography (v1.4)

## Vision
Rendro must handle complex internationalized text and advanced, multi-page data reporting tables without layout errors. These capabilities allow Rendro to graduate from basic statements to supporting enterprise-grade, localized, and highly dynamic document structures.

## Success Criteria
- A fully internationalized document (Arabic, CJK, Latin) renders deterministically with correct shaping and fallbacks.
- Multi-page tables with explicit fragmentation constraints break predictably across pages, repeating headers.
- Flow constraints like widow and orphan limits are strictly respected during pagination.

## Key Risks
- Pure-Elixir text shaping is complex and could cause performance regressions if not optimized.
- Table state spanning multiple pages introduces potential non-deterministic layout bugs.

## Proof Strategy
- Visual/Flow regressions will assert exact byte-for-byte or stable structural output.
- All new shaping and pagination features will have docs-contract tests demonstrating behavior.

## Verification Classes
- Deterministic byte-for-byte comparison.
- Structural layout tree inspection.

## Definition of Done
- All slices are executed, tested, and pass CI.
- `Rendro.Recipes.Invoice` updated to prove multi-page tables and i18n shaping.
- No headless Chrome or external C binaries were introduced.

## Requirement Coverage
- i18n shaping and font subsetting.
- Complex table pagination.
- Widow/Orphan text controls.

## Slices

- [ ] **S01: Pure-Elixir Font Subsetting Foundation** `risk:high` `depends:[]`
  > After this: A PDF can be generated that strictly embeds only the glyphs used in a "Hello World" string, validating the pure-Elixir font parser.
- [ ] **S02: Basic i18n Shaping & Font Fallbacks (Latin Extended + Arabic)** `risk:high` `depends:[S01]`
  > After this: An Arabic string renders with correct cursive ligatures and right-to-left directionality on a PDF page.
- [ ] **S03: Block-Level Flow Controls (Widow/Orphan Management)** `risk:medium` `depends:[]`
  > After this: A long paragraph at the end of a page correctly pushes an extra line to the next page to avoid an orphaned single line.
- [ ] **S04: Explicit Table Fragmentation Engine** `risk:medium` `depends:[S03]`
  > After this: A table explicitly configured with `{:fixed, ...}` splits perfectly across a page boundary, repeating headers on page two without silent truncation.
- [ ] **S05: Advanced Layout Verification & Recipe Updates** `risk:low` `depends:[S02, S04]`
  > After this: The canonical `Invoice` recipe can render Arabic customer details inside a multi-page table with perfect pagination and shaping.

## Boundary Map
- `S01` → `S02`: `S01` produces parsed font metrics and glyph maps; `S02` consumes them to execute complex shaping rules.
- `S03` → `S04`: `S03` establishes advanced block pagination primitives; `S04` consumes these primitives to implement table-row breaking semantics.
- `S02, S04` → `S05`: `S05` integrates the new `Rendro.Text` (shaping) and `Rendro.Table` (fragmentation) behaviors into canonical `%Rendro.Document{}` recipes.
