**Confidence:** HIGH (adoption case studies verified against live forum threads, hex.pm download APIs, GitHub workflows, and maintainer retrospectives)

## B. Launch-artifact features (in scope)

### Visual recipe gallery + self-rendered manual (Phase 86)
- All five recipes rasterized to PNGs embedded in README + HexDocs; images regenerated and **hash-checked in CI** (new docs-contract lane — gallery can't drift from the engine).
- Prawn-style `manual.pdf` generated by Rendro itself with published SHA-256: "this manual is byte-reproducible — verify it." A marketing claim no competitor (ChromicPDF/mudbrick/Typst wrappers) can copy.
- Presentation follows `prompts/Rendro Brand Book.txt`.

### Comparison/benchmark page (Phase 87)
- "Generating PDFs in Elixir without Chrome": cold start, RSS, container image size, dependency count vs ChromicPDF / pdf_generator / Typst-CLI. Claims bounded to checked-in scripts + results (proof culture as marketing).
- **Honest about where HTML→PDF wins**: arbitrary CSS layout, complex-script text — which truthfully advertises the v2.7 shaping demand gate.

### Livebook lane (Phase 87)
- One `.livemd` tutorial: invoice data → render → Kino inline preview → download. "Run in Livebook" badges via ExDoc. Executed in CI so it can't rot.

### Launch execution + demand instrumentation (Phase 88)
- ElixirForum #announcing thread (hook: 1.0 + stability contract + gallery), ElixirStatus post, awesome-elixir PR, genuine replies in the two demand threads, optional Show HN ("Deterministic PDF generation in pure Elixir — same input, same bytes").
- **Mobile viewer evidence as content**: record 2–4 high-traffic rows (iOS Files/Mail preview, Android Drive viewer × forms/signed surfaces). Expected mostly `explicit_deferral` — itself a publishable finding ("what mobile viewers actually do to your signed invoice"). Mobile pain is real: Adobe KB documents form data missing on mobile Reader; signature validation effectively absent on mobile.
- **Demand-gate definition** (closes the open circuit): concrete v2.7 shaping trigger — e.g. N non-self GitHub issues/forum asks + download threshold + first external contributor; ADOPTION.md signal ledger; GitHub Discussions/issue templates ("what document did you need that Rendro couldn't render?").

## C. Engine features (in scope)

### Drawn-path primitive (Phase 84) — table stakes
- Every comparator had graphics before TOC/charts: Prawn::Graphics (line/curve/rect/rounded_rect/polygon/ellipse, stroke/fill variants), ReportLab `graphics.shapes` (declarative Drawing of Rect/Line/Path data objects — **the model to copy**), Typst's drawing primitives (cetz ecosystem built on them).
- Rendro shape: declarative `%Rendro.Path{}` authored element — NOT a Prawn-style imperative canvas. Unlocks: table rules/borders/header bands (currently zero borders), Certificate `border:` frame, future charts.

### Pdfium render lane + golden-PNG harness (Phase 85)
- Prior art: **Typst** (renders refs with in-tree renderer, ≤20 KiB committed reference PNGs, runner requires a ref for visual output — the model); **fpdf2** (`assert_pdf_equal` on qpdf-normalized QDF — Rendro's determinism makes the bytes-level equivalent already trivial); **WeasyPrint** (ASCII-art pixel patterns for coarse geometry); **PDFium itself** (Skia Gold, multiple acceptable expectations); **pdf.js** (per-platform refs, known flaky — the expensive end to avoid).
- Two deliverables, one harness: (a) golden-PNG snapshot tests per layout feature; (b) machine-recorded viewer-evidence rows with new automated `viewer_kind`.

## D. Deferred features (designs documented for later)

- **Global text shaping (v2.7, demand-gated)**: thin honest slice = real UAX #9 bidi (pure Elixir, UCD conformance suite) + cluster-aware line breaking + Arabic/Hebrew vertical slice with named fonts + per-script matrix rows. NOT the caller-provides-pre-shaped-text workaround (the genre that burned Prawn/ReportLab users). Precedents: fpdf2 `set_text_shaping` (opt-in, soft dep, enumerated limitations — the template), ReportLab 4.4 experimental shaping, iText pdfCalligraph paid add-on.
- **TOC**: reserve-space design (see ARCHITECTURE.md) — fixed-height rows, fixed-width page-number column, `{{toc_page:<anchor>}}` tokens through existing substitution; TOC page count computable pre-pagination; **no fixpoint iteration ever** (Typst caps at 5 and ships stale numbers; ReportLab multiBuild raises after 10; LaTeX oscillates — all anti-models). Free byproduct: `/Outlines` bookmarks from the anchor map.
- **Charts**: native `%Rendro.Chart{}` lowering to Path+Text; decimal-based tick selection; **never SVG import** (svglib's issue tracker is a catalog of font/text failures; contex emits SVG). ReportLab chart DX ("extremely verbose", undocumented) is the cautionary tale — if/when built, ship few options, document every one.
- **Even/odd headers, section restart**: `suppress_on: :even | :odd` + `{{section_page_number}}`/`{{section_total_pages}}` tokens — near-trivial when demanded (Prawn repeaters precedent).
- **Multi-signature**: pyHanko supports DocMDP/FieldMDP/sequential incremental updates when a future milestone wants it; cheap hedge = a "counter-signing with external tools" recipe doc with `explicit_deferral` reason.
